# Cortex Capital - Deployment Guide

## Single Service Deployment (Render)

Backend serves the frontend as static files - one unified service.

### Prerequisites

1. Render account
2. GitHub repo connected to Render
3. Environment variables ready

### Custom Domain (Optional)

1. Go to Render service → Settings → Custom Domains
2. Add: `cortexcapital.com` (or your domain)
3. Add DNS records (Render provides values):
   - A record: `@` → Render IP
   - CNAME record: `www` → `cortex-capital.onrender.com`

### Environment-Specific URLs

**Development (local):**
- Frontend: `http://localhost:3000`
- Backend: `http://localhost:3001`

**Production (Render):**
- Both: `https://cortex-capital.onrender.com`

Frontend auto-detects environment and uses relative `/api/*` paths.

### Troubleshooting

**Build fails:**
- Check Render build logs
- Verify all dependencies in package.json
- Ensure frontend/package.json exists

**Frontend shows 404:**
- Check that `frontend/out/` directory was created during build
- Verify `fastifyStatic` is serving from correct path
- Check Render logs for static file errors

**API calls fail:**
- Verify DATABASE_URL is set correctly
- Check JWT_SECRET is generated
- Ensure CORS allows your domain

**Database connection fails:**
- Verify Render PostgreSQL is running
- Check DATABASE_URL has SSL enabled
- Test connection from Render shell

### Rollback

If deployment fails:

1. Go to Render Dashboard
2. Click service → Settings → Deploys
3. Find last working deploy
4. Click "Redeploy"

Or rollback via GitHub:
```bash
git revert HEAD
git push origin main
```

### Local Testing (production mode)

Test production build locally before deploying:

```bash
# Build frontend
cd frontend && npm run build && cd ..

# Start server in production mode
NODE_ENV=production npm start

# Visit http://localhost:3001 (serves both frontend + API)
```

### Continuous Deployment

Render auto-deploys on every push to `main` branch.

To disable:
- Render Dashboard → Service → Settings
- Build & Deploy → Auto-Deploy: OFF
